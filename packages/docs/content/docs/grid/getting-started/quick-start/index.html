Data Den is a plugin that extends the functionality of the library with table component.

NPM

NPM is a recommended way to work with Data Den and others libraries.

Make sure that you have Node.js installed. In order to verify that, open a command line and type node -v. If node is installed you should see it's version in a console.

1. Before starting the project make sure to install Node.js (LTS).

2. Run the following command to install the package via NPM:

{{}} {{}} npm install data-den-core {{< /twsnippet/code>}} {{< /twsnippet/wrapper>}}

3. Import the entire library or just individual modules:

{{}} {{}} import * as mdb from 'data-den-core'; // lib import { DataDen } from 'data-den-core'; // module {{< /twsnippet/code>}} {{< /twsnippet/wrapper>}}

4. Import Data Den stylesheet please use the following syntax:

CSS

{{}} {{}} @import 'data-den-core/dist/css/data-den.min.css'; {{< /twsnippet/code>}} {{< /twsnippet/wrapper>}}

JS

{{}} {{}} import 'data-den-core/dist/css/data-den.min.css'; {{< /twsnippet/code>}} {{< /twsnippet/wrapper>}}

Now, the library is ready to use and available via the global DataDen variable.

MDB GO / CLI

Create, deploy and host anything with a single command.

1. To start using MDB GO / CLI install it with one command:

{{}} {{}} npm install -g mdb-cli {{< /twsnippet/code>}} {{< /twsnippet/wrapper>}}

2. Log into the CLI using your MDB account:

{{}} {{}} mdb login {{< /twsnippet/code>}} {{< /twsnippet/wrapper>}}

3. Initialize a project and choose Data Den from the list:

{{}} {{}} mdb init data-den-core {{< /twsnippet/code>}} {{< /twsnippet/wrapper>}}

4. Install the dependencies (inside the project directory):

{{}} {{}} npm install {{< /twsnippet/code>}} {{< /twsnippet/wrapper>}}

5. Run the app:

{{}} {{}} npm start {{< /twsnippet/code>}} {{< /twsnippet/wrapper>}}

6. Publish when you're ready:

{{}} {{}} mdb publish {{< /twsnippet/code>}} {{< /twsnippet/wrapper>}}

Vite

Note: Make sure that you have Node.js version 16.17.0 + / 18.1.0 + installed. Please update if your package manager asks for it. If you are expecting issues with vite.config.js and you don't want to update the npm, try using the 4.3.9 version of vite.

Vite (French word for "fast", pronounced /vit/) is a build tool that aims to provide a faster and leaner development experience for modern web projects. Data Den can be imported in Vite applications according to the following procedure:

1. Create a new vite project and enter the newly created directory. Pick Vanilla framework and JavaScript variant. You can skip this step if you already have it installed.

{{< twsnippet/wrapper "terminal" "" "mobile" >}} {{< twsnippet/code active=true lang="Plaintext" type="terminal" >}} npm create vite@latest my-project cd my-project {{< /twsnippet/code >}}{{< /twsnippet/wrapper >}}

2. Install the Data Den package.

{{< twsnippet/wrapper "terminal" "" "mobile" >}} {{< twsnippet/code active=true lang="Plaintext" type="terminal" >}} npm install data-den-core {{< /twsnippet/code >}}{{< /twsnippet/wrapper >}}

3. Import Data Den stylesheets to your style.css file. You can also remove the default content of the style.css file if you want.

{{< twsnippet/wrapper "style.css" "" "mobile" >}} {{< twsnippet/code active=true lang="css" >}} @import 'data-den-core/dist/css/data-den.min.css'; {{< /twsnippet/code >}}{{< /twsnippet/wrapper >}}

4. Link the css file inside the index.html.

{{< twsnippet/wrapper "index.html" "" "mobile" >}} {{< twsnippet/code active=true lang="HTML" >}} ... ... {{< /twsnippet/code >}}{{< /twsnippet/wrapper >}}

5. Data Den should be connected properly to the vite app. If you type the code bellow, you will see the default vite index.html page (without any styles if you deleted them from style.css).

{{< twsnippet/wrapper "terminal" "" "mobile" >}} {{< twsnippet/code active=true lang="Plaintext" type="terminal" >}} npm run dev {{< /twsnippet/code >}}{{< /twsnippet/wrapper >}}

6. Now you can start working on your app. Since the package.json created by vite has a type set to module, we are going to use the ES. Lets try to import some code and see if everything is working correctly.

{{< twsnippet/wrapper "index.html,main.js" "" "mobile" >}} {{< twsnippet/code active=true lang="html" >}} Data Den App
